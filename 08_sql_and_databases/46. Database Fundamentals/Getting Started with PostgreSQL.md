## PostgreSQL Installation and Initial Setup on Pop!_OS

### Overview

This guide documents the complete installation and initial configuration process of PostgreSQL on Pop!_OS/Linux. It also covers connecting PostgreSQL with graphical database management tools such as DataGrip and pgAdmin 4.

The purpose of these notes is to:

* Maintain a reproducible PostgreSQL setup workflow
* Understand how PostgreSQL services operate on Linux
* Learn how GUI tools interact with a PostgreSQL server
* Build foundational database administration knowledge

---

## Installing PostgreSQL

### Step 1: Install PostgreSQL Common Utilities

```bash
sudo apt install -y postgresql-common
```

This package provides:

* PostgreSQL service management utilities
* Cluster management scripts
* Repository configuration support

---

### Step 2: Add PostgreSQL Official Repository

```bash
sudo /usr/share/postgresql-common/pgdg/apt.postgresql.org.sh
```

This script:

* Adds the official PostgreSQL APT repository
* Configures repository signing keys
* Enables installation of the latest PostgreSQL versions

---

## Verifying PostgreSQL Installation

The installation was verified using:

```bash
apt show postgresql
```

Output summary:

```text
Package: postgresql
Version: 16+257build1.1
Depends: postgresql-16
Description: object-relational SQL database
```

### Important Observation

The `postgresql` package itself is a metapackage.

It automatically installs the currently supported PostgreSQL version, which in this setup is:

```text
PostgreSQL 16
```

---

## Checking PostgreSQL Service Status

The service status was checked using:

```bash
sudo systemctl status postgresql
```

Observed output:

```text
Active: active (exited)
```

### Why Does PostgreSQL Show `active (exited)`?

This is normal behavior on Ubuntu/Pop!_OS systems.

The `postgresql.service` unit acts mainly as a wrapper service responsible for managing PostgreSQL clusters. The actual PostgreSQL database instance runs through cluster-specific services internally.

The important part is that:

* The service is enabled
* No failure occurred
* PostgreSQL clusters are operational

## Accessing PostgreSQL Using `psql`

### Switching to PostgreSQL System User

```bash
sudo -i -u postgres
```

The `postgres` Linux user is automatically created during PostgreSQL installation and owns the database processes.

---

### Starting PostgreSQL Interactive Shell

```bash
psql
```

Observed output:

```text
psql (16.13)
Type "help" for help.
```

This launches the PostgreSQL interactive terminal client.

## Connecting PostgreSQL with DataGrip

### About DataGrip

DataGrip is a professional database IDE developed by JetBrains.

It provides:

* SQL editor
* Database explorer
* Query execution tools
* Schema visualization
* Multi-database support

---

## Creating PostgreSQL Connection in DataGrip

### Connection Details Used

| Field          | Value                |
| -------------- | -------------------- |
| Name           | `postgres@localhost` |
| Driver         | PostgreSQL           |
| Host           | `localhost`          |
| Port           | `5432`               |
| User           | `postgres`           |
| Authentication | User & Password      |
| Database       | `postgres`           |

---

## Important Authentication Observation

Using:

```text
No Auth
```

did not work.

Therefore, setting a password for the PostgreSQL user was necessary.

---

## Setting PostgreSQL User Password

Inside `psql`:

```sql
ALTER USER postgres PASSWORD 'password';
```

This enables password-based authentication for GUI tools and external applications.

---

## JDBC Connection URL

Automatically generated by DataGrip:

```text
jdbc:postgresql://localhost:5432/postgres
```

### Breakdown

| Component     | Meaning                    |
| ------------- | -------------------------- |
| `jdbc:`       | Java Database Connectivity |
| `postgresql:` | PostgreSQL JDBC driver     |
| `localhost`   | Local machine              |
| `5432`        | Default PostgreSQL port    |
| `postgres`    | Target database            |

```text
http://127.0.0.1/pgadmin4
```

```text
http://localhost/pgadmin4
```

---

## Connecting PostgreSQL with pgAdmin

A new server named:

```text
main-server
```

was created inside pgAdmin.

The same PostgreSQL credentials and localhost configuration were used.

---

## Important Observation About Shared Databases

After connecting pgAdmin to PostgreSQL, the database created earlier in DataGrip (`trial_db`) was also visible in pgAdmin.

This confirms an important concept:

> DataGrip and pgAdmin are only client applications.
> The actual databases are stored and managed by the PostgreSQL server itself.

Both tools simply connect to the same PostgreSQL instance.

## Understanding PostgreSQL Internal Components

### Casts

Defines type conversion rules between different data types.

---

### Catalogs

Internal PostgreSQL metadata tables storing database information.

---

### Event Triggers

Triggers executed on database-level events.

---

### Extensions

Additional PostgreSQL functionality modules.

Examples:

* `uuid-ossp`
* `pgcrypto`
* `postgis`

---

### Foreign Data Wrappers

Allows PostgreSQL to connect with external data sources.

---

### Languages

Programming languages supported inside PostgreSQL procedures.

Examples:

* SQL
* PL/pgSQL

---

### Publications

Used in logical replication systems.

---

### Schemas

Logical containers used to organize tables and database objects.

Default schema:

```text
public
```

---

### Subscriptions

Receives replicated data from publications.
